# TailscaleOHOS 原生 UI 设计基线

本文把 2026-07-15 保存的 35 份 HarmonyOS 官方设计与组件文档转化为本项目可执行的设计约束。原始页面来自华为开发者联盟；`upstream-sync.htm` 与设计无关，未纳入。

## 核心美学

HarmonyOS 的美学不是“给界面加更多效果”，而是让内容、空间与交互处于和谐而清楚的关系中：

- 以人为本：信息清晰、操作直觉、反馈及时，用户无需理解技术实现。
- 平衡共生：前景与背景、品牌色与中性色、动与静之间保持克制的对比。
- 空间连续：沉浸光感用于跨越内容层和悬浮交互层，表达 Z 轴关系，而不是装饰所有内容容器。
- 秩序优先：通过语义色、字体层级、8vp 栅格、统一圆角和一致动效建立结构。
- 一次开发，多端适配：移动优先，并在断点处改变导航和内容编排，而不是等比放大手机界面。

## 信息架构

- 一级结构仅保留“主页”和“设置”两个互斥页签。
- 主页承载核心任务：查看状态、登录/连接、出口节点、设备。
- 设置承载低频偏好：网络设置、沉浸光感四档、账户资料与管理、版本信息、工程诊断。
- 一级页使用强调型标题；不显示返回按钮。标题简短，说明文字只补充当前决策所需的信息。
- 普通手机使用悬浮底部页签；宽度达到 840vp 时改为侧边页签。内容宽度达到 600vp 后可使用双栏。

## 原生组件策略（API 23）

- 页面根导航：`HdsTabs`，手机使用 API 23 的 `barFloatingStyle`，内容区禁止横滑切页以避免与页面手势冲突。
- 标题：一级页采用原生标题栏语义与系统字体，不自绘 Web 风格 Header。
- 图标：优先 HarmonyOS Symbol；图标与文字共享基线和语义色。
- 设置项：使用原生 `Toggle`、`Button`、列表/分组容器及系统资源色。
- 沉浸光感选择使用原生 `Select` 下拉按钮，当前项由系统勾选标识；组件位于设置页末尾，避免四个状态按钮抢占首屏。
- 登录和连接进入异步等待时，在原按钮内显示 20vp `LoadingProgress`，保持按钮尺寸稳定并暂时禁用重复点击。
- 页面 `Scroll` 必须绑定各自 `Scroller` 到 `HdsNavigation`；标题使用 `FREE` 模式随内容连续缩小到左上角，禁用大面积标题栏模糊背板。
- 设置页使用 4vp 原生自动滚动条：进入与滚动时显示，无操作后由系统收起。
- 按钮：优先使用 API 23 原生 `ButtonStyleMode`、`ControlSize` 与 `ButtonRole`；主操作使用强调型、普通操作使用普通型、退出登录使用警示角色，不手工复刻系统配色和圆角。
- 同一界面只保留一个最高强调级主操作；按钮文案使用动宾结构。等待状态在按钮内反馈，避免额外闪现加载弹窗。
- 状态按钮必须成组、清晰表达选中与未选中；单选组只允许一个当前项。
- 底部页签已承担一级导航，因此不叠加工具栏；当前仅有一个核心连接动作，不使用多动作的核心操作栏。
- 系统颜色：优先 `sys.color.font_primary`、`font_secondary`、`icon_primary`、`background_primary`、`comp_background_primary` 等语义 Token，自动适配深浅色。
- 字体：使用系统默认 HarmonyOS Sans，不指定自定义字体；正文随系统字体缩放。

## 视觉 Token

### 间距与栅格

- 基础网格：8vp；微小图标和细节允许 4vp。
- 手机页面边距：16vp；中等宽度：24vp；大宽度：32vp。
- 普通组件间距：8vp；有明确边界的组件间距：16vp；卡片/分组间距：12vp。
- 主次文本间距：2vp。
- 栅格：`xs/sm/md/lg` 分别以 2/4/8/12 列为基础，内容最大宽度 2220vp。

### 圆角

- 4vp：标签、角标。
- 8vp：图片、图标等常规展示元素。
- 16vp：中层内容容器与提示。
- 20vp：按钮、菜单等高频交互元素。
- 32vp：弹窗、半模态等高层浮层。
- 同层同类必须统一；视觉层级越高，圆角可越大。

### 色彩

- 页面使用不透明的系统背景色；不以透明背景承担页面层级。
- 品牌蓝只用于主操作、选中态和关键状态，避免大面积铺色。
- 文本按 `primary / secondary / tertiary / fourth` 表达优先级。
- 成功、警告、危险状态使用系统语义色；不得只依靠颜色表达状态，必须同时有文字或形状反馈。

## 沉浸光感

- 默认使用“自适应”，跟随设备默认光感；用户可选“精美 / 轻柔 / 流畅 / 自适应”四档，但由同一套原生材质定义响应，不复制四套界面。
- 顶部悬浮组件建议 `ULTRA_THIN` 并配合渐变模糊；底部悬浮组件建议 `THIN` 并配合与页面背景一致的渐变颜色层。
- 非常驻、任意位置弹出的组件可使用 `THICK` 保障可读性。
- 本项目只在主按钮和悬浮页签使用光感。内容分组使用系统实色，避免“满屏玻璃卡片”。
- 使用组件默认光效；若需要品牌光色，采用低透明度、高亮度颜色。
- 悬浮页签高度 56vp，图标 24vp；两枚页签在小于 600vp 时宽 168vp，在不小于 600vp 时宽 184vp。由 `HdsTabs` 按断点自动计算，不写死宽度。
- 底部渐变蒙层应比页签上边缘高 16vp，并避让系统导航条。

## 动效与反馈

- 动效服务于层级、状态和因果，不作为装饰。
- 颜色变化约 100ms，开关反馈约 150ms，列表变化约 200ms；短距离不超过 250ms，长距离不超过 350ms。
- 优先使用可打断的弹性曲线和系统状态动效；运动路径保持一致。
- 点击必须即时反馈，等待过程显示明确状态；页签选中使用轻微 Symbol 弹跳或系统默认反馈。

## 文案与可访问性

- 重要信息先说，句子短、直接、友好；少用工程术语和“失败/错误”等强负向词。
- 操作按钮使用动宾结构，例如“连接 Tailscale”“退出登录”。
- 错误提示同时说明用户下一步可以做什么。
- 触控目标、文本缩放、横竖屏、深色模式和系统安全区均不得破坏核心任务。

## 本项目的验收条件

- API 23 为 compatibleSdkVersion 和 targetSdkVersion。
- 首页可完成登录、连接/断开与出口节点选择；不出现网络设置开关或未验证的 MagicDNS 兼容性宣称。
- 设置页可调整网络选项、切换四档光感，并查看账户头像、账户资料、应用版本和 Tailscale 内核版本；工程诊断默认折叠。
- 手机为悬浮底栏，宽屏为侧边页签；600vp 以上内容不再保持狭长单列。
- 浅色/深色模式均可读，无固定白色卡片破坏系统主题。
- UI 探针、布局截图和真机构建安装均通过。
